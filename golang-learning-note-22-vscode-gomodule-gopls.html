go语言的vscode， golang module 和 gopls的一些笔记

21 December 2019

vscode， golang module 和 gopls的一些笔记

  1. 按照之前的学习笔记http://www.noteanddata.com/golang-learning-note-17-dependency-management.html, go语言的依赖管理现在应该是go modules
  2. go语言的IDE里面，我试用了atom, vscode, goland, 其中goland最好， 保持了jetbrains系列产品的高水准， 但是需要收费， 所以自己电脑上就没有买. atom更多的是一个编辑器， 虽然有一堆插件， 但是就算配置了插件以后， 还是和goland和vscode差距比较大， 所以准备自己电脑上主要使用vscode
  3. 但是试用了一下vscode和go module, 默认设置有比较多问题， 比如import就有问题，
    需要先安装一个gopls
go get golang.org/x/tools/gopls@latest

然后在setting里面加上这句话

"go.useLanguageServer": true

参考资料：
- vscode官方wiki https://github.com/Microsoft/vscode-go/wiki/Go-modules-support-in-Visual-Studio-Code
- gopls的官方wiki https://github.com/golang/tools/blob/master/gopls/doc/user.md

  1. 不过就算安装了以后， 还是有一些问题，有一些函数的go to implementation没有用， 感觉和goland还是有差距

gopls的一些概念

第一次接触到gopls， 所以简单学习了一下gopls的概念
1. gopls是Language Server Protocol https://microsoft.github.io/language-server-protocol/specifications/specification-3-14/的一个实现，
2. gopls是一个官方支持的，希望成为所有编辑器的一个backend， 来实现编辑器/ide的各种特性， 虽然现在支持还是十分有限

  1. 参考资料https://github.com/golang/tools/blob/master/gopls/doc/design.md
    这些参考资料的设计文档还是写的非常不错， 清晰的讲了设计的目标， 背景， 哪些不是gopls的设计目标， 现有的解决方案， 技术难点和特点等， 非常值得学习。
