linkedin面试题目 leetcode 150 Evaluate Reverse Polish Notation

  1. 输入逆波兰表达式，先出现两个操作数，后出现操作符
  2. 输入的操作符有+-*/
  3. 求解这个表达式

刷题感悟

写这个题目是因为在刷几个计算器的题目时候，，http://www.noteanddata.com/leetcode-224-Basic-Calculator-java-solution-note.html
http://www.noteanddata.com/leetcode-227-Basic-Calculator-II-java-solution-note2.html
http://www.noteanddata.com/leetcode-772-Basic-Calculator-III-google-interview-problem-java-solution-note.html
看到有一些做法是把中序表达式转化成后序表达式， 也就是逆波兰表达式(Reverse Polish Notation), 然后直接求解。
感觉那样的做法虽然代码写的长了一点， 时间空间复杂度也高一些，但是逻辑非常清晰直接，而且非常通用。其实是一个非常好的通用做法， 而其他做法通常依赖于对各种情况和状态的仔细分析，而且容易出错。

解题思路分析

逆波兰表达式的求解非常直接，就是一个堆栈，遇到数就入栈， 遇到操作符就把栈顶的两个元素拿出来运算， 然后把结果入栈

java题解代码

public int evalRPN(String[] tokens) {
    if(tokens.length == 0) {
        return 0;
    }
    Stack<String> stack = new Stack<>();
    for(String token: tokens) {
        if(token.equals("+") || token.equals("-") || token.equals("*") || token.equals("/")) {
            int v2 = Integer.valueOf(stack.pop());
            int v1 = Integer.valueOf(stack.pop());

            if(token.equals("+")) {
                stack.push(String.valueOf(v1+v2));
            }
            else if(token.equals("-")) {
                stack.push(String.valueOf(v1-v2));
            }
            else if(token.equals("*")) {
                stack.push(String.valueOf(v1*v2));
            }
            else if(token.equals("/")) {
                stack.push(String.valueOf(v1/v2));
            }
        }
        else {
            stack.push(token);
        }
    }
    return Integer.valueOf(stack.pop());
}

时间复杂度

O(N)

空间复杂度

O(N)
