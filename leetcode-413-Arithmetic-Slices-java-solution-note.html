题目–Leetcode 413. Arithmetic Slices

https://leetcode.com/problems/arithmetic-slices/description/
输入一个整数数组，定义其中的一个长度至少为3的连续子数组为a[i], a[i+1], a[i+2]….a[j],
使得其中的所有相邻两个数的差都是相等的（等差数列），
要求返回总共有多少个等差数列.

A = [1, 2, 3, 4]

return: 3, for 3 arithmetic slices in A: [1, 2, 3], [2, 3, 4] and [1, 2, 3, 4] itself.

解题分析

  1. 因为是需要连续子数组，所以不是特别复杂，遍历的时候保存两个相邻数的差，和前面的差做比较， 如果一样的就继续，直到遇到不一样的，
  2. 然后计算这个长度的等差数列总共有多少个连续子数组也是等差数列，基本上就是把从3到N的可能性都取一遍

java代码

版本1

public int numberOfArithmeticSlices(int[] A) {
    if(A.length <= 2) return 0;
    boolean valid = true;
    int begin = 0;
    int total = 0;
    for(int i = 2; i < A.length; ++i) {
        if(A[i] - A[i-1] == A[i-1]-A[i-2]) {
            if(!valid) {
                begin = i-2;
            }
            valid = true;
        }
        else {
            if(valid) {
                int len = i - begin;
                total += calcCount(len);
            }
            valid = false;
        }
    }    
    if(valid) {
        int len = A.length - begin;
        total += calcCount(len);
    }
    
    return total;
}

public int calcCount(int len) {
    return (len-1) * (len-2)/2;
}

其中，计算长度的函数，一开始是这样写的, 后来才改成上面的形式，

public int calcCount(int len) {
    int count = 0;
    for(int i = 3; i <= len; ++i) {
        if(len-i+1 >= 0) {
            count += len-i+1;
        }
    } 
    return count;
}

推算公式如下：
i=len –> 1
i=len-1 –> 2
i=len-2 –> 3
…
i=3 –> len-3+1 = len-2

时间复杂度

数组遍历了一次， 时间复杂度是O(N)

空间复杂度

只需要常量， 空间复杂度是O(1)

其他想法

这里要求的是连续子数组， 如果不是要求连续子数组， 可以任意取几个数做子数组， 那就会变成另外一道题目，
这个思路有待后续再慢慢想。
