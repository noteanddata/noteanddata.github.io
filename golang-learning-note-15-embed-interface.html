go语言接口的继承和嵌套

01 September 2019

go语言里面的接口继承

go语言里面没有直接的接口继承关键字， 接口的继承是通过声明同样的方法来做的
比如下面这个例子， bird这个接口里面也声明了printName()和fly()方法， 那么, birdImpl既可以是animal, 也可以是flyable, 也可以是bird

package main

import "fmt"

type animal interface {
	printName()
}

type flyable interface {
	fly()
}

type bird interface {
	//animal
	//flyable
	printName()
	fly()
}

type birdImpl struct {
	name string
}

func (a birdImpl) printName() {
	fmt.Println("this is ", a.name)
}
func (a birdImpl) fly() {
	fmt.Println("I am flying: ", a.name)
}

func main() {
	test1()
}

func test1() {
	b1 := birdImpl{}
	testAnimal(b1)
	testf(b1)
	testbird(b1)
}

func testAnimal(a animal) {
	a.printName()
}

func testf(f flyable) {
	f.fly()
}

func testbird(b bird) {
	b.printName()
	b.fly()
}

go语言里面的接口嵌套

当然， go语言里面还可以用接口嵌套， 在一个接口里面声明另外一个接口， 那么，对应的方法就自动继承过来了

package main

import "fmt"

type animal interface {
	printName()
}

type flyable interface {
	fly()
}

type bird interface {
	animal
	flyable
}

type birdImpl struct {
	name string
}

func (a birdImpl) printName() {
	fmt.Println("this is ", a.name)
}
func (a birdImpl) fly() {
	fmt.Println("I am flying: ", a.name)
}

func main() {
	test1()
}

func test1() {
	b1 := birdImpl{}
	testAnimal(b1)
	testf(b1)
	testbird(b1)
}

func testAnimal(a animal) {
	a.printName()
}

func testf(f flyable) {
	f.fly()
}

func testbird(b bird) {
	b.printName()
	b.fly()
}

其中， fmt.Fprintf需要的是一个io.Writer
https://golang.org/pkg/fmt/#Fprintf

package main

import (
    "fmt"
    "log"
    "net/http"
)

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hi there, I love %s!", r.URL.Path[1:])
}

func main() {
    http.HandleFunc("/", handler)
    log.Fatal(http.ListenAndServe(":8080", nil))
}

而http.ResponseWriter也是实现了 Write([]byte) (int, error)方法，所以也是一个io.Writer
