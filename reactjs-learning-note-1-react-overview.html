24 December 2019

reactjs的基本概念

  1. reactjs是facebook搞出来的一个前端框架， 主要思路是模块化
  2. JSX语法
    主要语法是JSX语法， 就像下面这样, 这里是把表示ui的元素和javascript结合在一起了， 所以是javascript的一种扩展。
const element = <h1>Hello, world!</h1>;

或者这样

const name = 'Josh Perez';
const element = <h1>Hello, {name}</h1>;

ReactDOM.render(
  element,
  document.getElementById('root')
);

这个思路本身就很奇怪了，因为整个软件业一直以来的金科玉律就是显示和逻辑实现分离, 从设计模式到各种技术实现都是按照这个标准来的， 现在突然来一个说法说要把显示和逻辑混合在一起， 那一定是很多人要说不好的， 不过facebook搞出react的人显然也不是菜鸟，人家专门有一个视频讲这个 https://www.youtube.com/watch?v=x7cQ3mrcKaY。 简单的来说， 是因为这些ui元素和逻辑本身就是紧密结合的，所以，react的方式是做一个一个独立的组件， 这些组件里面有ui元素， 也有逻辑脚本。

  1. 元素的渲染
    如果要把定义好的JSX元素显示在屏幕上， 那么需要调用ReactDOM.render方法
const element = <h1>Hello, world</h1>;
ReactDOM.render(element, document.getElementById('root'));

  1. 元素的不可变的(immutable)
    任何一个元素创建以后， 所有属性和子元素都不能修改， 要修改UI只能创建一个新的元素， 然后调用ReactDOM.render方法

  2. Virtual DOM的概念
    那么问题来了， 如果在界面上创建一个很复杂的日历时间控件， 假设会显示到秒， 那么每秒都需要更新一下UI， 如果每次都对DOM做一个很复杂的渲染的话， 效率肯定非常低下， 但是React有一个Virtual DOM的概念， 每次创建元素以后，这个都在Virtual DOM的概念里面， 最后真正更新DOM, 会对比DOM上有哪些元素是需要真正更新的， 然后只更新那部分

  3. Components的概念
    在react里面， 定义一个只有一个参数props的javascript函数就是定义了一个Components

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

或者用ES6的语法, 写一个类, 继承React.Component函数， 实现render()方法

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

有了Component以后， 就可以在其他JSX语法里面引用这些Component, 比如这样

const element = <Welcome name="Sara" />;

完整的代码如下

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

const element = <Welcome name="Sara" />;
ReactDOM.render(
  element,
  document.getElementById('root')
);
  1. Component的组合
    有了Component以后呢， 可以互相组合成更复杂的Component, 这些新的Component还可以继续被组合, react就是通过创造一个完整的， 内聚的， 同时包含UI和逻辑的组件， 来达到代码的复用。UI组件通常会变的非常复杂， 通过这种复用机制，可以提高工程效率
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

function App() {
  return (
    <div>
      <Welcome name="Sara" />
      <Welcome name="Cahal" />
      <Welcome name="Edite" />
    </div>
  );
}

ReactDOM.render(
  <App />,
  document.getElementById('root')
);
  1. Component的props是read-only的
    在react的设计里面， prop本身是read-only的， 不能被修改
    举例， Clock就可以这样设计
function Clock(props) {
  return (
    <div>
      <h1>Hello, world!</h1>
      <h2>It is {props.date.toLocaleTimeString()}.</h2>
    </div>
  );
}

function tick() {
  ReactDOM.render(
    <Clock date={new Date()} />,
    document.getElementById('root')
  );
}

setInterval(tick, 1000);
