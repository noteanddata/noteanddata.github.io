npm基本概念

npm是node package manager的简称， 是javascript的package management工具
在nodejs上，作为一个推荐的package management管理工具, 安装 nodejs的时候， npm就会被自动安装

比如我之前用nvm安装了nodejs 12.14, 然后就可以看到本地已经安装了对应的npm版本, 和
https://nodejs.org/en/download/releases/是对应的

$node --version
v12.14.0
$ npm --version
6.13.4

npm安装模块

  1. 使用npm install就可以安装模块
npm install $module_name

比如, 要安装express模块

npm install express

  1. 当然， 安装完以后我们需要知道这个模块在哪里
    可以使用npm root得到当前的模块安装位置, 比如默认是在~/node_modules
    如果指定了npm install -g express, 那就会全局安装， 会安装在~/.nvm/versions/node/v12.14.0/lib/node_modules
    同样， 全局安装的位置是通过npm root -g得到的

  2. npm init可以初始化一个npm模块， 这样会在当前目录下生成一个package.json, 这时候在这个目录下面运行npm install express, 会在当前目录下面生成一个node_modules目录， 然后安装在这里

npm清理模块

如果想要清理模块， 可以使用下面的命令， 或者干脆直接删除整个node_modules目录. 通常， 如果对nodejs或者npm进行版本升级的时候， 第三方库通常会有很多不兼容的， 可能需要先清理， 然后重新安装.

npm cache clean
npm install

这时候运行npm install， 其实就是根据package.json里面的dependencies来安装依赖

其他一些有用的npm命令

后面4个命令是跟着module_name

npm list
npm list express
npm info express
npm search express
npm install sax@0.1.1
