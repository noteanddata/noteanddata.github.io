Leetcode 462. Minimum Moves to Equal Array Elements II解题笔记

19 November 2018

题目 Leetcode 462. Minimum Moves to Equal Array Elements II

  1. 输入一个整数数组, 定义一个move操作是把其中一个数加1或者减1
  2. 求需要的最少的move操作，使得整个数组的长度是一样的

解题思路分析

  1. 从小规模样本出发， 假设所有的数都一样， 那么就返回0
  2. 如果只有两个数，
    [1,2]–> 1
    [1,3]–> 2
    [1,4]–> 3
    [1,10]–> 9

注意到这里的几个例子， move一定有多种选择， 但是效果都是一样的。
比如[1,10], 无论是最后都是1或者2或者5或者10， 需要都步骤都一样，
说明两个数的步骤就是两个数的差。

那就遍历一下数组，分别假设move到其中到某一个数，（就是把所有的差都累计起来（注意这里需要绝对值)）,
然后取最小都累计和就好， 因为最后的结果一定可以是其中的一个数.

注意这个问题直接取平均值是不行的， 比如
[1,1,1,1,1,100]

java代码

public int minMoves2(int[] nums) {
    long min = Integer.MAX_VALUE;
    for(int i = 0; i < nums.length; ++i) {
        long diffsum = 0;
        for(int j = 0; j < nums.length; ++j) {
            diffsum += Math.abs((long)nums[j] - (long)nums[i]);
        }
        min = Math.min(min, diffsum);
    }    
    return (int)min;
}

时间复杂度

两层循环， O(N^2)

空间复杂度

只使用了常数， 所以是O(1)

其他思考

这个解法只是暴力解，并不是最优解， 还有O(N*lgN), 甚至O(N)的解法， 有待后续学习
