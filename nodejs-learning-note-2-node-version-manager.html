nodejs学习笔记2–使用node version manager做nodejs的版本管理

版本依赖和管理

版本依赖在任何一个语言里面都是大麻烦， 一来因为一些功能的缺乏，导致新版本不停的在加新特性， 二来大量的第三方的库和框架，可能对特别版本有要求， 当然， 另外第三方库之间的依赖冲突又是另外一个问题。
nodejs的版本很多，前端技术更新也比较快，有一些框架和第三方的库对版本也有依赖， 为了方便多版本并存， 可以使用node version manager来管理多个不同的版本.

nodejs的版本管理–node version manager

nodejs的版本管理使用node version manager, 简称nvm

安装和使用

  1. 在mac下用下面的命令可以安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.2/install.sh | bash

默认安装后会写入到~/.bashrc, 如果~/.bash_profile里面没有引用到~/.bashrc的话， 在~/.bash_profile里面加入下面一行

source ~/.bashrc
  1. 使用
    然后开一个新的terminal就可以使用, 输入nvm可以验证安装是否成功
$nvm

Node Version Manager (v0.35.2)

然后就可以nvm install安装

$ nvm install 12.14.0
Downloading and installing node v12.14.0...
....
$ node --version
v12.14.0
