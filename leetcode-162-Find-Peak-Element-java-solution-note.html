leetcode 162 Find Peak Element题二分解题笔记

11 May 2019

题目 leetcode 162 Find Peak Element

  1. 输入一个整数数组， 求任意一个peak的元素的index
  2. 所谓peak的意思是说a[i-1] <= a[i] >= a[i+1]
  3. 假设最左边和最右边都是负无穷

解题思路分析

  1. 直接遍历当然可以， 问题是能不能用更好的做法？ 比O(N)更好的基本上只有二分了
  2. 我自己是万万没有想出二分的， 并且理解为什么二分正确也搞了很久。
    看了一圈我觉得核心的逻辑在于最左边和最右边有负无穷的存在，才可以导致这个题目可以用二分

  3. 二分的基本逻辑如下：
    a. 任意找一个中点， 如果有a[mid] > a[mid+1], 那么，我们可以直接从左边开始找？ 因为最左边有一个负无穷， 那么mid左边（包含mid）的数要么有比mid小的，要么有mid大的， 无论哪种情况，都可以有一个peak
    b. 同理， 如果是另外一种情况， 那么就可以从mid+1开始向右边找， 无论右边的数比左边大还是小，都可以找到一个peak
    c. 这里二分需要注意的是，左右切分的时候， 需要把当前较大的这个数包括进去。 否则会漏掉一个值.

java题解代码

public int findPeakElement(int[] nums) {
    return search(nums, 0, nums.length-1);
}

int search(int[] nums, int from, int to) {
    if(from == to) return from;
    int mid = from + (to-from)/2;
    if(nums[mid] > nums[mid+1]) {
        return search(nums, from, mid);    
    }
    return search(nums, mid+1, to);
}

时间复杂度

O(lgN)

空间复杂度

O(1)
