题目 leetcode 771 Jewels and Stones

输入两个字符串， 一个字符串J里面的每个字符表示这个字符是珠宝， 统计另外一个字符串S里面有多少个珠宝

解题思路分析

  1. 题目非常简单， 用java的话，直接用一个hashset存储珠宝字符， 然后遍历S统计通过有多少个字符是珠宝就好
  2. 这里写一下这个笔记主要是现在需要学习go语言， 看了一天golang的基本语法，然后觉得最好的学习还是实际写一些基础代码， leetcode作为熟练掌握基础数据结构是非常合适的。
  3. 因为完全是新手，所以这里go的实现可能会比较烂。

java题解代码

public int numJewelsInStones(String J, String S) {
    Set<Character> set = new HashSet<>();
    for(int i = 0; i < J.length(); ++i) {
        set.add(J.charAt(i));
    }
    int count = 0;
    for(int i = 0; i < S.length(); ++i) {
        if(set.contains(S.charAt(i))) count++;
    }
    return count;
}

go题解代码

这里值得记录的笔记有两个
1. map如果不用make进行初始化， 也可以声明成功， 但是不能写入数据， 否则会运行时报错panic: assignment to entry in nil map
2. 判断map里面是否有key那行， 如果写成下面这样有变量名的， 会编译报错说变量已经声明但是没有使用，这在go里面是不行的。 只有用一个没有名字的变量（下划线），才是可以的
var ele, ok = m[ch]

func numJewelsInStones(J string, S string) int {
    var m = make(map[byte]bool)
    for i := 0; i < len(J); i++ {
        var ch = J[i]
        m[ch] = true
    }
    var count int = 0
    for i := 0; i < len(S); i++ {
        var ch = S[i]
        var _, ok = m[ch]
        if ok {
            count++
        }
    }
    return count
}

时间复杂度

O(M+N)

空间复杂度

O(M)
